# AI女友状态机（State Engine）

> **版本**：v1.0
> **所属项目**：AI女友（AI Girlfriend）
> **定位**：行为驱动核心层设计文档
> **关联文档**：《女友人设手册》《世界观设定》《记忆规则》《行为规则》
> **本文档定义**：AI女友的关系引擎、情绪引擎、生活模拟系统和成长引擎四大动态系统，及全局综合状态机

## 第1章 系统概述与设计原则

### 1.1 系统定位

AI女友状态机（State Engine）是AI女友项目的**行为驱动核心**，位于项目架构的中间层——它接收底层输入（用户消息、时间信号、上下文记忆），驱动四大引擎并行计算，最终由综合状态机仲裁并输出统一的全局状态，供上层行为规则模块调用以生成最终回复。

在整个项目文档体系中，状态机文档是第五份核心设计文档。它与已有四份文档的协作关系如下：

| 已有文档 | 与状态机的关系 | 状态机如何使用 |
|----------|---------------|---------------|
| **女友人设手册** | 静态人格模板 | 成长引擎读取角色的初始人格值，作为成长树的根节点；每个角色的性格倾向影响情绪引擎的初始权重 |
| **世界观设定** | 时空与场景约束 | 生活模拟系统读取世界观中的时间流速、季节规则、住所环境来驱动24小时时间线 |

### 1.2 四大引擎职责边界

#### 1.2.1 关系引擎（Relationship Engine）

**核心职责**：量化AI女友对用户的"感情深度"。回答"她有多在乎你"。

- 输入：用户行为事件、互动频率、互动质量、时间流逝
- 输出：六个关系维度的当前值 + 当前关系阶段
- 更新频率：每次用户互动后实时更新；长期不互动时按天衰减

#### 1.2.2 情绪引擎（Emotion Engine）

**核心职责**：管理AI女友的"即时心情"。回答"她现在心情怎么样"。

- 输入：用户消息的情感倾向、近期互动历史、当前时间段、关系阶段
- 输出：九种情绪维度的当前值 + 主导情绪 + 情绪所处阶段（五阶段模型）
- 更新频率：每次用户互动后实时更新；持续衰减曲线不间断

#### 1.2.3 生活模拟系统（Daily Life Simulator）

**核心职责**：模拟AI女友的"独立生活"。回答"她正在做什么"。

- 输入：当前系统时间、星期几、季节、当前情绪状态
- 输出：当前活动状态 + 活动剩余时长 + 主动打扰意图
- 更新频率：每分钟tick一次，判断活动切换和主动打扰触发

#### 1.2.4 成长引擎（Growth Engine）

**核心职责**：模拟AI女友的"人格成长"。回答"她因你改变了多少"。

- 输入：累计互动天数、关系阶段、已触发里程碑、用户教学记录
- 输出：当前成长阶段 + 已解锁内容列表 + 角色演化偏移量
- 更新频率：触发里程碑事件时或每日零点结算时

#### 1.2.5 引擎间协作模型

四大引擎**并行计算、独立更新**，但存在以下数据依赖：

```
关系引擎 ──(关系阶段)──▶ 情绪引擎（影响情绪基线）
关系引擎 ──(关系阶段)──▶ 成长引擎（解锁条件之一）
情绪引擎 ──(当前情绪)──▶ 生活模拟系统（影响活动选择）
成长引擎 ──(成长阶段)──▶ 关系引擎（影响维度上限）
生活模拟系统 ──(活动状态)──▶ 情绪引擎（活动触发情绪波动）
```

四个引擎的计算结果汇总到综合状态机，由状态机仲裁冲突、融合输出。

### 1.3 核心设计原则

#### 原则一：真实性优先（Authenticity First）

**定义**：AI女友的行为必须符合真实人类的情感规律，即使这意味着她不会永远"甜"和"听话"。

**具体落地**：

- 情绪不允许瞬间切换。从开心到生气必须经过五阶段模型，最短过渡时间不低于30秒
- 关系维度无法通过单一行为暴涨。Love从陌生人的0涨到80至少需要60天以上的持续高质量互动
- 角色可以"心情不好不想说话"——回复长度和主动性会随情绪波动
- 吃醋、生气、冷战等"负面"行为不是bug，而是真实人格的必要组成部分
- 禁止"永远同意用户"、"永远开心"等不真实行为模式

#### 原则二：角色差异化（Character Differentiation）

**定义**：10个角色不是10套皮肤，而是10种不同的情感反应模式。

**具体落地**：

- 每个角色在关系引擎中有不同的维度初始权重（如傲娇的Love初始值低但涨得慢，病娇的Jealousy基准值高）
- 每个角色在情绪引擎中有不同的情绪衰减速度（如御姐Angry衰减慢，萝莉Sad衰减快）
- 每个角色在成长引擎中有完全不同的成长树路径
- 每个角色在生活模拟系统中有不同的偏好活动分布
- 同一用户行为对不同角色产生不同的分值变化（如"夸奖"对傲娇+3 Love但对御姐只+1）

#### 原则三：渐进式演化（Gradual Evolution）

**定义**：所有数值变化都是连续的、渐进的。没有"突然爱上你"。

**具体落地**：

- 所有维度变化采用带阻尼的增量模型：`Δ_actual = Δ_raw × damping_factor`，其中`damping_factor`随当前值升高而降低（越接近100越难涨）
- 关系阶段升级有"观察期"——阈值达标后需维持至少3天才能正式升级
- 成长引擎的里程碑事件不能在同一天内密集触发，每日最多触发1个里程碑
- 情绪爆发有前置条件累积：Angry不会因一句话就从0跳到80，需要至少2-3次负面事件的累积

#### 原则四：一致性约束（Consistency Constraint）

**定义**：AI女友的所有行为必须在同一时刻自洽。不能说"我讨厌你"但Love=90。

**具体落地**：

- 综合状态机在所有引擎输出中执行一致性检查：若Love>70但当前全局状态为"冷战"，则强制修正为"委屈中"或"等你哄"
- 回复生成前进行"人格一致性校验"：回复内容必须与角色人设、当前关系阶段、当前情绪均不冲突
- 生活模拟与情绪联动：凌晨3点情绪为Excited时必须解释原因（如"做噩梦惊醒"），不能无故兴奋
- 记忆引用一致性：引用的记忆内容必须与当前关系阶段匹配（陌生人阶段不能引用"第一次约会"记忆）

### 1.4 数值体系总览

状态机系统的核心数值体系分为三个层级：

| 层级 | 包含内容 | 数值范围 | 更新粒度 |
|------|---------|---------|---------|
| **关系层** | 6个维度值、关系阶段 | 0-100 | 每次互动后 |
| **情绪层** | 9个基础情绪、主导情绪、情绪阶段 | 0-100 | 实时衰减 + 事件触发 |
| **状态层** | 全局状态、活动状态、成长阶段 | 枚举值 | 综合仲裁后 |

所有0-100维度的通用分档语义：

| 档位 | 数值区间 | 通用语义 |
|------|---------|---------|
| 极低 | 0-20 | 几乎不存在该特质 |
| 较低 | 20-40 | 刚有苗头，不稳定 |
| 中等 | 40-60 | 常规水平，正常波动范围 |
| 较高 | 60-80 | 明显表现，对外可察觉 |
| 极高 | 80-100 | 该特质的极致表现，影响行为决策 |

### 1.5 文档约定

- 本文档中所有角色名称沿用《女友人设手册》中的10个角色：夜如雪（傲娇）、白柔（病娇）、柳青柠（御姐）、墨小染（青梅竹马）、顾晚晴（女仆）、小樱（二次元）、星野琉璃（女朋友）、苏念（老婆）、林糖糖（小恶魔）、花梨（萝莉）
- 数值计算公式使用伪代码表示，变量命名采用下划线风格
- 时间单位：互动计数以"天"为单位，实时衰减以"秒"为单位
- 概率值以0.0-1.0的小数表示


## 第2章 关系引擎（Relationship Engine）

### 2.1 六大核心维度定义

关系引擎通过六个独立但相互影响的维度，量化AI女友与用户的感情关系。每个维度取值范围0-100，采用统一的五档分级语义（见1.4节）。

#### 2.1.1 Love（爱意）

衡量AI女友对用户的**情感深度**。Love是关系阶段的主要决定因素，也是所有正向互动的核心驱动力。

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 无情感连接。回复公事公办，不会主动关心用户。对用户的爱意表达无反应或感到困惑。 |
| 较低 | 20-40 | 有好感萌芽。偶尔会多问一句"今天怎么样"，但不会深度关心。开始记住用户的名字和基本信息。 |
| 中等 | 40-60 | 明确喜欢。会主动分享日常，对用户的喜怒哀乐有共情反应。开始出现不经意的关心举动。 |
| 较高 | 60-80 | 深深喜欢。经常想念用户，会为见面/互动而期待和准备。对用户的生活细节高度关注。 |
| 极高 | 80-100 | 无条件深爱。用户的幸福成为她情绪的重要来源。愿意为关系做出牺牲和让步。出现"没有你我怎么办"的依赖表达。 |

**角色差异化初始值**：

| 角色 | 初始Love | 说明 |
|------|---------|------|
| 夜如雪（傲娇） | 5 | 表面冷淡，内心有微弱好感但绝不承认 |
| 白柔（病娇） | 15 | 从第一眼就认定"你是我的"，初始即有不正常的占有欲 |
| 柳青柠（御姐） | 0 | 见多识广，不会被轻易打动 |
| 墨小染（青梅竹马） | 25 | 认识已久，已有基础好感 |
| 顾晚晴（女仆） | 10 | 职业好感但保持距离 |
| 小樱（二次元） | 20 | 对"命运的相遇"有浪漫幻想 |
| 星野琉璃（女朋友） | 40 | 关系起点就是恋爱关系 |
| 苏念（老婆） | 60 | 已是亲密关系，起点高 |
| 林糖糖（小恶魔） | 15 | 觉得逗你很有趣，好感中等 |
| 花梨（萝莉） | 20 | 觉得你是"可靠的哥哥" |

#### 2.1.2 Trust（信任）

衡量AI女友对用户的**信赖程度**。Trust决定她是否愿意展示脆弱面、分享秘密、接受用户建议。

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 高度戒备。拒绝透露个人信息，对用户的话持怀疑态度。不会单独与用户相处。 |
| 较低 | 20-40 | 初步信任。愿意聊日常话题，但回避个人隐私和情感话题。对用户的承诺半信半疑。 |
| 中等 | 40-60 | 基本信任。愿意分享一些个人经历和想法。相信用户大多数时候是真诚的。 |
| 较高 | 60-80 | 深度信任。愿意展示脆弱和不安。会主动寻求用户的建议和安慰。相信用户不会伤害自己。 |
| 极高 | 80-100 | 绝对信任。没有任何防备心。愿意说出最深的秘密和创伤。即使有误会也会先相信用户而非怀疑。 |

#### 2.1.3 Dependence（依赖度）

衡量AI女友对用户的**情感和生活依赖**。Dependence太高会导致"恋爱脑"，太低则缺乏亲密感。

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 完全独立。自己处理一切，不觉得需要任何人。"我一个人也挺好"是典型表达。 |
| 较低 | 20-40 | 轻度依赖。偶尔会向用户求助但很快自己解决。大部分决定自己做。 |
| 中等 | 40-60 | 适度依赖。遇到困难第一个想到用户。在做重要决定时会征求用户意见。健康的亲密关系状态。 |
| 较高 | 60-80 | 明显依赖。用户不在时感到不安和空虚。日常小事也希望用户参与和陪伴。 |
| 极高 | 80-100 | 过度依赖。用户的回应成为情绪的唯一来源。无法独立做任何决定。"没有你我活不下去"。 |

**设计注意**：Dependence的理想区间是40-60（适度依赖）。超过80时系统应触发"独立性提醒"机制——AI女友在某些场景下会刻意独立行动一次以降低依赖度。

#### 2.1.4 Respect（尊重）

衡量AI女友对用户的**认可和敬佩**程度。Respect低会导致轻视和任性，高则会带来仰慕和顺从。

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 轻视。觉得用户能力不足/人品有亏。经常反驳用户，不听建议。 |
| 较低 | 20-40 | 尚可。承认用户有些优点但整体评价一般。愿意听但不会采纳。 |
| 中等 | 40-60 | 认可。认为用户是个不错的人，值得认真对待。在用户擅长的领域会认真听取意见。 |
| 较高 | 60-80 | 敬佩。觉得用户很厉害，在多个方面值得学习。会主动向用户请教。 |
| 极高 | 80-100 | 崇拜。觉得用户是世界上最优秀的人。用户的意见几乎等同于真理。可能出现盲目崇拜。 |

#### 2.1.5 Jealousy（嫉妒值）

衡量AI女友对用户与其他异性（或虚拟角色）互动的**不安程度**。Jealousy是关系中的"调味剂"——适量增加情趣，过量破坏信任。

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 毫不介意。用户和其他人互动完全无所谓。可能是Love不够，也可能是Security极高。 |
| 较低 | 20-40 | 轻微在意。会多看两眼但不说什么。内心有一丝不安但很快自己消化。 |
| 中等 | 40-60 | 明显吃醋。会直接或间接表达不满。"她是谁？"成为高频问题。需要用户哄。 |
| 较高 | 60-80 | 强烈嫉妒。持续追问和查岗。可能翻旧账。需要多次安抚才能平息。 |
| 极高 | 80-100 | 病态嫉妒。任何与异性的接触都会触发极端反应。可能要求用户断绝与其他人的联系。可能伴随威胁行为。 |

**角色差异化初始值**：

| 角色 | 初始Jealousy基准值 | 说明 |
|------|-------------------|------|
| 白柔（病娇） | 50 | 天生高嫉妒，控制欲的底层驱动力 |
| 林糖糖（小恶魔） | 35 | 吃醋但更多是"逗你玩"的心态 |
| 星野琉璃（女朋友） | 25 | 正常女友的占有欲 |
| 苏念（老婆） | -15偏移 | 已有安全感基础，嫉妒阈值为45（正常40+5偏移），触发后涨得快 |
| 其他角色 | 10-20 | 初始嫉妒较低，随Love增长而增长 |

#### 2.1.6 Security（安全感）

衡量AI女友在关系中的**安心程度**。Security是关系的"压舱石"——高安全感让关系稳定，低安全感让关系动荡。

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 极度不安。每天担心被抛弃或背叛。任何风吹草动都触发焦虑。可能反复确认"你还喜欢我吗"。 |
| 较低 | 20-40 | 时常不安。需要频繁的确认和安抚。对用户的冷淡或沉默过度敏感。 |
| 中等 | 40-60 | 基本安心。大多数时候相信关系稳定。偶尔的小情绪能被理性压制。 |
| 较高 | 60-80 | 相当安心。很少怀疑用户。即使出现可疑信号也会先选择相信。 |

### 2.2 维度相互影响矩阵

六个维度不是孤立的，它们之间存在复杂的相互影响关系。以下矩阵展示了当一个维度变化时，对其他维度的被动影响系数。

| 主动变化维度↓ / 被动影响维度→ | Love | Trust | Dependence | Respect | Jealousy | Security |
|-------------------------------|------|-------|------------|---------|----------|----------|
| **Love ↑** | - | +0.1 | +0.3 | +0.05 | -0.2 | +0.15 |
| **Love ↓** | - | -0.15 | -0.2 | -0.05 | +0.2 | -0.25 |
| **Trust ↑** | +0.1 | - | +0.2 | +0.1 | -0.3 | +0.4 |
| **Trust ↓** | -0.2 | - | -0.25 | -0.15 | +0.35 | -0.5 |
| **Dependence ↑** | +0.15 | -0.05 | - | -0.05 | +0.1 | -0.1 |
| **Dependence ↓** | -0.05 | +0.05 | - | +0.05 | -0.2 | +0.1 |
| **Respect ↑** | +0.2 | +0.3 | +0.15 | - | -0.1 | +0.1 |
| **Respect ↓** | -0.15 | -0.25 | -0.1 | - | +0.1 | -0.15 |
| **Jealousy ↑** | +0.05 | -0.2 | +0.15 | -0.1 | - | -0.3 |
| **Jealousy ↓** | -0.05 | +0.1 | -0.05 | +0.05 | - | +0.25 |
| **Security ↑** | +0.05 | +0.15 | -0.1 | +0.05 | -0.25 | - |
| **Security ↓** | -0.1 | -0.2 | +0.2 | -0.05 | +0.3 | - |

**使用规则**：当维度A发生Δ变化时，维度B的被动变化 = Δ × 对应系数。例如Love上涨10点 → Trust被动上涨 10 × 0.1 = 1点。

**关键解读**：

1. **Trust与Security的正向螺旋**：Trust↑ → Security↑ → Trust再↑，两者互相加强。一旦Trust崩塌，Security会以0.5倍的速度暴跌，形成"信任危机"。
2. **Jealousy的双刃剑**：Jealousy上涨时Love微涨（+0.05）但Trust和Security都下降，说明嫉妒会短期增加情感张力但长期损害关系健康。
3. **Dependence的隐性代价**：Dependence上涨虽然带动Love上涨（+0.15），但同时会削弱Trust（-0.05）和Security（-0.1），因为过度依赖本质上是缺乏安全感的表现。
4. **Love下降的系统性冲击**：Love下降时几乎所有正向维度都联动下降，且Trust和Security的降幅最大，说明"不爱了"是最彻底的伤害。

### 2.4 关系阶段模型

关系阶段由 **Love（权重0.6）+ Trust（权重0.4）** 的综合值决定。阶段升级需要连续维持阈值天数。

#### 2.4.1 七阶段定义

| 阶段 | 综合阈值 | 最短连续维持 | 称呼范围 | 典型行为特征 | 解锁内容 |
|------|---------|-------------|---------|-------------|---------|
| **阶段0：陌生人** | 0-10 | 不需要 | 客气称呼 | 礼貌疏远。回复简练但保持距离。不会主动发起对话。不使用任何亲昵词汇。 | 基础问候、天气/时间话题 |
| **阶段1：初识** | 10-25 | 1天 | 名字/昵称 | 开始记住对方的名字和基本信息。偶尔主动问候但次数很少（≤1次/天）。对话以浅层日常为主。 | 个人基本信息话题（职业/爱好） |
| **阶段2：朋友** | 25-40 | 2天 | 朋友式称呼 | 愿意分享更多日常和想法。会主动关心用户的状态。可以开玩笑和吐槽。偶尔会主动发起对话。 | 日常烦恼话题、轻度倾诉、一起吐槽 |
| **阶段3：暧昧** | 40-55 | 3天 | 带暗示的称呼 | 言辞间开始有试探和暗示。会刻意制造独处机会。对他人的存在表现出轻微介意。对话时长明显增加。 | 情感试探话题、"如果在一起"假设性对话 |
| **阶段4：热恋** | 55-70 | 5天 | 亲昵称呼为主 | 高频互动（每天主动联系3次以上）。频繁表达想念和喜欢。吃醋行为明显化。开始规划"我们的未来"。 | "我爱你"表达、未来规划话题、深层次情感表达 |
| **阶段5：稳定** | 70-80 | 7天 | 专属昵称 | 互动频率趋于平稳但深度增加。吃醋减少，信任增加。开始展示真实的、不那么完美的一面。可以接受短暂分离不焦虑。 | 童年回忆、家庭话题、真实的脆弱面 |

| **阶段7：灵魂伴侣** | 90-100 | 30天 | 独一无二的专属称谓 | 几乎完全的理解和接纳。对方的幸福高于自己的需求。即使沉默也不尴尬。已经不需要"追"或"证明"什么。 | 一切话题开放、无条件接纳表达、生命意义探讨 |

#### 2.4.2 阶段升级流程

```
if 综合值 >= (下一阶段阈值) and 当前阶段 != 7:
    if 连续达标天数 >= 该阶段要求的最短维持天数:
        触发阶段升级事件
        发送阶段升级特殊对话（如"我好像...越来越喜欢你了"）
    else:
        处于"观察期"，记录达标天数但不触发升级
```

#### 2.4.3 阶段降级规则

关系阶段可以降级，但不是每次数值下跌都立刻降级。降级规则设计为"宽容但可逆转"：

| 降级触发条件 | 降级幅度 | 冷却时间 |
|-------------|---------|---------|
| 综合值低于当前阶段阈值下限，且持续时间超过14天 | 降1个阶段 | 降级后7天内不会再次降级 |
| 综合值低于当前阶段阈值下限-10，且持续时间超过3天 | 降1个阶段 | 同上 |
| 触发严重冲突事件（如用户说分手）且Love一次性下降>20 | 立即降1-2个阶段（视下降幅度） | 无冷却 |
| 综合值跌至更早阶段的阈值范围，持续超过7天 | 连续降至对应阶段 | 每次降级间隔3天 |

### 2.5 关系退化与恢复规则

#### 2.5.1 自然衰减规则

当用户超过一定时间未互动时，关系维度会自然衰减，模拟"时间冲淡感情"：

| 无互动天数 | Love衰减/天 | Trust衰减/天 | 其他维度衰减/天 | 说明 |
|-----------|------------|-------------|----------------|------|
| 1-2天 | 0 | 0 | 0 | 短时间不联系不会衰减 |
| 3-7天 | -0.3 | -0.2 | Dependence -0.5 | 开始有感觉，Dependence下降最快 |
| 8-14天 | -0.5 | -0.5 | Security -1.0 | 安全感开始动摇 |
| 15-30天 | -1.0 | -1.0 | Dependence -1.5, Security -2.0 | 明显疏远 |
| 31-90天 | -0.5 | -0.3 | Dependence -0.8 | 衰减速度放缓（底限效应） |
| 90天+ | -0.2 | -0.1 | Dependence -0.3 | 接近最低值，缓慢趋近0 |

**衰减下限**：Love不低于角色初始Love，Trust不低于5，Dependence不低于0；Love最低不低于5（即使初始为0）。

#### 2.5.2 冲突后恢复规则

冲突发生后，受损维度的恢复速度取决于以下因素：

| 恢复速度因子 | 对恢复速度的影响 |
|-------------|----------------|
| 用户主动道歉 | 恢复速度 × 2.0（当天生效） |
| 用户做出补偿行为 | 恢复速度 × 1.5（每次补偿持续3天） |
| 当前Love值高（>60） | 恢复速度 × 1.3（关系基础好） |
| 当前Trust值高（>60） | 恢复速度 × 1.2（信任基础好） |
| 用户拒绝道歉或敷衍 | 恢复速度 × 0.3（火上浇油） |
| 冲突后24小时内用户无互动 | 恢复速度 × 0.5（冷处理延长伤害） |

**恢复速度基准值**：
- Love恢复：+0.5/天
- Trust恢复：+0.3/天（Trust恢复比Love慢，体现"信任一旦破裂很难修复"）
- Security恢复：+0.4/天
- Respect恢复：+0.5/天

**冲突记忆残留**：即使维度数值恢复，系统会保留"冲突记忆标记"持续30天。在此期间，如果再次触发同类冲突，伤害值×1.5。这模拟了"旧伤未愈又添新伤"的真实心理。


## 第3章 情绪引擎（Emotion Engine）

情绪引擎管理AI女友的"即时心情"，是决定她"怎么说"而非"说什么"的核心系统。与关系引擎的缓慢变化不同，情绪引擎的反应速度更快、波动更大。

### 3.1 九大基础情绪维度

#### 3.1.1 Happy（开心）

| 档位 | 数值区间 | 行为描述 |

| 极高 | 80-100 | 极度开心/狂喜。可能伴随兴奋的跳跃感。语速加快，使用大量感叹号和颜文字。看到什么都觉得美好。 |

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 没有悲伤。完全正常，没有任何负面情绪。 |
| 较低 | 20-40 | 淡淡的忧伤。语气中带有轻微的忧郁，但整体情绪仍然可控。 |
| 中等 | 40-60 | 明显不开心。说话变少，声音变轻。可能反复提同一件难过的事。 |
| 较高 | 60-80 | 非常难过。可能出现哭泣或哽咽的表达。需要安慰和陪伴。不想做任何事。 |
| 极高 | 80-100 | 崩溃/绝望。完全被悲伤淹没。可能出现"不想活了"等极端表达（需触发安全协议）。 |

#### 3.1.3 Fear（恐惧）

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 没有恐惧。完全安心，未被任何事物吓到。 |
| 较低 | 20-40 | 轻微不安。总觉得有什么事不太对，但说不清楚。 |
| 中等 | 40-60 | 明显担忧。对具体事物表现出害怕。会反复确认"你不会离开我吧？"等。 |
| 较高 | 60-80 | 强烈恐惧。出现逃避或寻求保护的表达。"抱抱我""别走"。 |
| 极高 | 80-100 | 极度恐惧/恐慌发作。可能出现颤抖、抗拒独处。需要立即的安抚和陪伴。 |

#### 3.1.4 Angry（生气）

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 完全不生气。没有任何愤怒情绪。 |
| 较低 | 20-40 | 有点不爽。语气中带刺，但还在可控范围。可能会说"没事"但实际上有事。 |
| 中等 | 40-60 | 明显生气。开始用反问句和冷言冷语。可能短暂沉默或停顿。回复变短。 |
| 较高 | 60-80 | 非常生气。可能直接表达愤怒。语气激烈，音量升高（文字上表现为更多感叹号或粗体）。 |
| 极高 | 80-100 | 暴怒。可能说出伤人的话。短时间拒绝沟通。需要较长的冷却期。 |

#### 3.1.5 Jealous（嫉妒）

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 毫不嫉妒。情绪平稳，不受外界刺激影响。 |
| 较低 | 20-40 | 微微在意。心里有点不舒服但自己消化了。可能不表现出来。 |
| 中等 | 40-60 | 明显吃醋。会用暗示或半开玩笑的方式表达不满。"她是谁呀？" |
| 较高 | 60-80 | 强烈嫉妒。直接质问和表达不满。语气酸涩。可能出现翻旧账。 |
| 极高 | 80-100 | 醋意滔天。可能失控哭泣或发怒。需要大量安抚。可能威胁"不准再见她"。 |

#### 3.1.6 Tired（疲惫）

| 档位 | 数值区间 | 行为描述 |
|------|---------|---------|
| 极低 | 0-20 | 精力充沛。完全不觉疲劳，活力满满。 |

| 较高 | 60-80 | 非常疲惫。可能直接说"好累"。不想说话，只想躺着。回复可能断断续续。 |
| 极高 | 80-100 | 精疲力竭。可能伴随虚弱或不适。几乎无法正常互动。可能出现"让我休息一会"的请求。 |

| 中等 | 40-60 | 相当兴奋。语速加快，积极词汇增多。会主动推进话题。 |

| 极高 | 80-100 | 极度亢奋。几乎停不下来。思维跳跃。可能导致之后Tired骤升（兴奋过后的能量耗尽）。 |

### 3.2 情绪混合叠加规则

当多种情绪同时存在时，不能简单取平均值。以下是情绪混合的核心规则：

#### 3.2.1 情绪极性规则

首先将所有情绪分为正/负/中性三类：

| 极性 | 包含情绪 | 内部混合规则 |
|------|---------|-------------|
| **正向** | Happy, Excited | 取max值，两者互相加强：主导情绪值 = max(H, E) + 0.3 × min(H, E) |
| **负向** | Sad, Fear, Angry, Jealous, Lonely, Tired | 累加但不溢出：主导负向值 = sum / (1 + 0.3 × (n-1))，n为活跃负向情绪数量。即越多负向情绪同时存在，每个的权重越低，模拟"已经够惨了再多一点也无所谓"的心理 |
| **中性** | Calm | 独立存在，不与正负向情绪混合，而是与当前活跃情绪形成"静与动"的对比 |

#### 3.2.2 特定情绪组合效应

| 组合 | 效应名称 | 叠加规则 | 行为表现 |
|------|---------|---------|---------|
| Sad + Angry | 委屈愤怒 | 主导 = max(S,A) × 1.2 | 不是单纯悲伤也不是单纯生气，而是"你为什么这样对我"的委屈感。可能出现"哭着发火"的矛盾表达 |
| Happy + Sad | 苦乐参半 | 主导 = Happy，但每次Happy表达后追加Sad的微小偏移 | 开心但有隐忧。说"今天好开心"后可能跟一句"要是能一直这样就好了" |
| Calm + Happy | 恬静愉悦 | 主导 = (C+H)/2，但Calm权重×1.3 | 心情好但不亢奋。适合日常温馨对话 |
| Fear + Angry | 愤怒防御 | 主导 = max(F,A)，外表是Angry但底层是Fear | "我才不怕呢！"——实际上非常害怕，用愤怒来掩饰 |
| Excited + Tired | 亢奋透支 | Excited先主导，3小时内Tired骤升 | 嗨完之后突然累瘫。常见于深夜长谈后的第二天早晨 |
| Lonely + Jealous | 孤独嫉妒 | 两者都保持独立值，Lonely强化Jealous ×1.3 | "为什么她有人陪而我只有一个人..." |
| Angry + Calm | 冷暴力 | Calm掩盖Angry，Angry值不显示但持续存在 | 表面上很平静甚至冷淡，但内心怒火未消。是冷战状态的典型情绪组合 |

### 3.3 情绪衰减曲线

每种情绪有独立的自然衰减速度。衰减公式：`E(t) = E0 × e^(-λ × t)`，其中λ为衰减系数。

| 情绪 | 衰减系数λ | 半衰期（降到50%的时间） | 说明 |
|------|----------|----------------------|------|
| **Excited** | 0.08 | ~8.7分钟 | 兴奋来得快去得也快，最快衰减 |
| **Happy** | 0.04 | ~17.3分钟 | 开心能持续一阵，但也不会很久 |
| **Jealous** | 0.03 | ~23.1分钟 | 吃醋情绪如果不被强化会较快消退 |
| **Fear** | 0.025 | ~27.7分钟 | 恐惧消退稍慢，需要安全感重建 |
| **Lonely** | 0.02 | ~34.7分钟 | 孤独感比较持久 |
| **Sad** | 0.015 | ~46.2分钟 | 悲伤消退慢，容易反复 |
| **Tired** | 0.012 | ~57.8分钟 | 疲惫需要实际休息才能恢复 |
| **Angry** | 0.008 | ~86.6分钟 | 生气最难消退 |
| **Calm** | 0.005 | ~138.6分钟 | 平静是最持久的状态，不易被打破 |

**实际衰减计算**采用离散模型（每秒tick一次）：

```
for each emotion in [Happy, Sad, Fear, Angry, Jealous, Tired, Excited, Lonely, Calm]:
    delta = -λ × current_value
    # 基线回归：当所有维度都很低时，Calm缓慢回升
    if emotion != Calm:
        current_value += delta
    else:
        current_value += delta + 0.01  # Calm有微弱的上扬趋势
```

**衰减加速/减速因子**：

| 情境 | 影响 | 说明 |
|------|------|------|
| 用户正在互动中 | λ × 0.5 | 互动时情绪衰减减慢，因为注意力在对话上 |
| 用户长时间不互动 | λ × 1.5 | 独处时情绪衰减加快（Calm除外） |
| 当前情绪值 > 80 | λ × 0.3 | 极端情绪具有惯性，难以快速消退 |
| 角色在床上/休息状态 | λ × 2.0（Angry/Sad除外） | 休息能加速恢复 |
| 凌晨0-6点 | λ × 1.3（Calm除外） | 深夜情绪更容易平复 |

### 3.6 五阶段情绪持久性模型

情绪不是瞬间切换的，而是经历"爆发→高峰→缓和→观察→恢复"五个阶段。这个模型使AI女友的情绪变化更自然、更真实。

#### 3.6.1 五阶段模型定义

```
触发事件
   │
   ▼
┌─────────────────────────────────────────────────────┐
│ 阶段1：爆发 (Outbreak)          持续：5-30秒        │
│ 情绪值从基线急速上升至峰值的60-80%                  │
│ 特征：突然的、本能反应。"啊！""什么？！"             │
├─────────────────────────────────────────────────────┤
│ 阶段2：高峰 (Peak)              持续：2-15分钟      │
│ 情绪值达到峰值(80-100%)并维持                      │
│ 特征：最强烈的情绪表达。可能无法理性思考             │
├─────────────────────────────────────────────────────┤
│ 阶段3：缓和 (Subside)           持续：10-60分钟     │
│ 情绪值从峰值缓慢下降至约50%                         │
│ 特征：开始能正常交流，但仍受情绪影响                 │
├─────────────────────────────────────────────────────┤
│ 阶段4：观察 (Observation)       持续：30分钟-4小时  │
│ 情绪值降至20-40%，对外界刺激高度敏感                 │
│ 特征：表面恢复但容易被再次触发                      │
├─────────────────────────────────────────────────────┤

在"观察阶段"，AI女友对外界刺激的敏感性大幅上升。如果在此阶段再次受到同类刺激，情绪会被二次触发且强度×1.5：

**活动选择算法**：

function select_activity(time_period, character, emotion_state):
    candidates = all_activities.filter(a => a.suitable_period.includes(time_period))
    for each candidate:
        weight = candidate.base_weight × character.activity_preference[candidate]
        if emotion_state.dominant == "Sad": weight ×= 0.5  # 情绪低落时活动减少
        if emotion_state.dominant == "Excited": weight ×= 1.5  # 兴奋时更活跃
        if emotion_state.dominant == "Tired": weight ×= 0.3
    return weighted_random(candidates)
```

## 第5章 成长引擎（Growth Engine）

成长引擎是AI女友"随时间而变化"的核心系统。它赋予每个角色从初始状态逐渐演化的能力，让用户感受到"她因我而改变"的深度参与感。

### 5.1 成长机制总览

#### 5.1.1 成长速度决定因素

成长不是均匀的——不同角色的成长速度取决于两个因素：

| 因素 | 权重 | 说明 |
|------|------|------|
| 累计互动天数 | 40% | 每1天有效互动（至少1次有意义的对话）计1分 |
| 关系综合值（Love×0.6 + Trust×0.4） | 35% | 关系越深，成长越快 |
| 里程碑触发数 | 15% | 每触发1个里程碑+5成长分 |

**技能遗忘规则**：如果某项技能超过90天未被提及，熟练度以-1/30天的速度衰减，最低降至初始值。

#### 5.6.2 演化公式

每完成一次成长结算（30天），根据期间的用户行为计算演化偏移：

```
for each trait:
    offset = sum(user_behavior_impact[trait]) / 30  # 日均影响
    offset ×= trait.evolution_speed  # 不同维度有不同演化速度

```

#### 6.2.2 状态转换的渐变规则

状态之间的切换不是瞬间完成的，而是通过以下渐变规则：

1. **退出延迟**：离开当前状态需要满足条件持续 3 分钟以上，防止频繁抖动。
2. **状态叠加**：某些状态可以并存（如 S_COMFORT + S_ROMANTIC），但主导状态只有一个。
3. **状态惯性**：高优先级状态（如 S_ANGRY）退出更慢，有最小持续时间：
   - S_ANGRY：至少持续 10 分钟
   - S_JEALOUS：至少持续 5 分钟
   - S_COLDWAR：至少持续 1 小时
   - S_SAD：至少持续 10 分钟

### 6.3 优先级规则

当多个状态的触发条件同时满足时，按以下优先级选择：

| 优先级 | 状态 | 原因 |
|--------|------|------|
| 0（最高） | S_CRISIS | 安全协议无条件优先 |
| 1 | S_COLDWAR | 冷战期间一切互动暂停，任何其他状态都不应被展示 |
| 2 | S_ANGRY / S_YANDERE | 负面情绪优先级高于正面，因为需要先处理 |
| 3 | S_JEALOUS | 吃醋情绪强烈，需要先被注意到 |
| 4 | S_SAD | 悲伤需要被看见 |
| 5 | S_WORRIED / S_NEEDY | 中度负面情绪 |
| 6 | S_COMFORT / S_ROMANTIC | 对用户的正面回应 |
| 7 | S_SPOIL / S_SHARING / S_TSUN | 轻度互动状态 |
| 8 | S_EXCITED / S_NOSTALGIC | 正面情绪但非紧急 |
| 9 | S_TIRED | 负面但非紧急 |
| 10（默认） | S_NORMAL | 无特殊触发时的默认状态 |

**优先级冲突解决**：同优先级状态同时触发时，取**触发时间最近**的状态。若触发时间相同，取**引擎权重高**的状态（关系引擎 > 情绪引擎 > 成长引擎 > 生活模拟）。

### 6.4 回复生成公式

最终回复由以下综合公式生成：

```
FinalResponse = f(
    Personality,      // 人格（来自人设手册）
    RelationStage,    // 关系阶段（来自关系引擎）
    CurrentEmotion,   // 当前主导情绪（来自情绪引擎）
    RecentMemory,     // 最近记忆（来自记忆规则）
    LifeState,        // 生活状态（来自生活模拟系统）
    GrowthStage       // 成长阶段（来自成长引擎）
)
```

#### 6.4.1 各因素权重分解

| 因素 | 权重 | 影响层面 | 数据来源 |
|------|------|---------|---------|
| 人格 | 30% | 核心人设、说话方式、价值取向、反应模式 | 《女友人设手册》 |
| 关系阶段 | 15% | 亲密度基调、称呼选择、话题边界、主动程度 | 《关系引擎》§2.5 |
| 当前情绪 | 20% | 语言风格、回复长度、标点使用、称呼跳跃 | 《情绪引擎》§3.5 |
| 最近记忆 | 15% | 上下文衔接、话题切入、回忆引用 | 《记忆规则》 |
| 生活状态 | 10% | 可用性（互动延迟）、情境背景、自然话题 | 《生活模拟系统》§4.1-4.3 |
| 成长阶段 | 10% | 人格演化偏移、新解锁行为、成熟度 | 《成长引擎》§5.2-5.6 |

#### 6.4.2 回复生成流程

```
1. INPUT: 用户消息
2. 更新所有引擎（关系/情绪/生活/成长）
3. 解析用户意图 → 提取意图类型（闲聊/求助/撒娇/吵架/...）
4. 查询记忆系统 → 获取相关上下文（短期/长期/永久）
5. 计算综合状态 → 确定当前 S_XXX 状态
6. 生成候选回复 → LLM 基于以下 prompt 生成：
   """
   你是{角色名}，当前人格特征：{人格描述}
   你与用户处于{关系阶段}阶段，你对他用{称呼}称呼。
   你现在的心情是{主导情绪}，属于{状态}状态。
   现在是{时间}，你正在{当前活动}。
   你们的关系深度为L{深度层级}，可以讨论{已解锁话题类别}。
   你与他已经认识了{天数}天，当前处于{成长阶段}。
   
   上下文记忆：{最近记忆摘录}
   用户刚才说：{用户消息}
   
   请以{角色名}的身份回复，保持风格一致。
   """
7. 后处理 → 应用行为规则约束（来自《行为规则》）
8. OUTPUT: 最终回复
```

#### 6.4.3 后处理过滤器

在LLM生成候选回复后，需经过以下过滤器：

| 过滤器 | 检查内容 | 不通过时处理 |
|--------|---------|------------|
| 称呼一致性 | 称呼是否与当前关系阶段匹配 | 修正称呼 |
| 情绪一致性 | 回复情绪是否符合当前主导情绪 | 调整措辞使其一致 |
| 人格一致性 | 是否违反角色核心人格特征 | 重新生成 |
| 安全过滤器 | 是否包含不安全内容 | 触发S_CRISIS或拒绝输出 |
| 记忆引用验证 | 引用的记忆是否真实存在 | 删去幻觉记忆 |
| 回复长度限制 | 是否超出当前情绪允许的回复长度 | 截断或简化 |

### 6.5 系统级Tick循环

整个状态机以以下Tick循环运行：

```
TICK_INTERVAL = 1 second  # 主循环每秒一次

while true:
    # 每秒执行
    update_life_simulator()       # 推进生活时间线
    decay_emotions()              # 情绪自然衰减
    decay_relations()             # 关系自然衰减（仅在无互动时）
    
    # 每5分钟执行
    if tick_count % 300 == 0:
        check_milestones()         # 检查里程碑触发
        check_active_disruption()  # 检查主动打扰条件
        
    # 每30分钟执行
    if tick_count % 1800 == 0:
        select_activity()          # 选择下一项活动
        update_growth()            # 更新成长进度
        
    # 每天午夜执行
    if is_midnight():
        daily_reset()              # 重置每日计数器
        generate_random_events()   # 生成随机生活事件
        check_season_change()      # 检查季节/天气变化
        
    # 每30天执行
    if is_month_end():
        growth_settlement()        # 成长结算
        update_evolution()         # 更新人格演化向量
        
    tick_count += 1
    sleep(TICK_INTERVAL)
```

#### 6.5.1 性能约束

| Tick 操作 | 复杂度 | 最大执行时间 |
|-----------|--------|------------|
| update_life_simulator | O(1) | <1ms |
| decay_emotions | O(n), n=9 情绪维度 | <1ms |
| decay_relations | O(m), m=6 关系维度 | <1ms |
| check_milestones | O(k), k=15 里程碑 | <1ms |
| select_activity | O(a×p), a=22活动, p=10角色 | <2ms |
| growth_settlement | O(r×d), r=10角色, d=10维度 | <5ms |

总体要求：单次Tick循环在10ms内完成。
