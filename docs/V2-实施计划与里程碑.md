## 风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|:---:|----------|
| LLM API 延迟过高，多角色并发导致响应慢 | 体验差 | 中 | 阶段四做批量调用优化；考虑降级为角色串行回复 |
| WebSocket 连接不稳定 | 消息丢失 | 中 | 实现心跳 + 自动重连 + 消息去重 |
| 角色回复质量参差不齐 | 群聊混乱 | 低 | 阶段二调优 System Prompt；阶段四参数调优 |
| 10+ 角色同屏渲染性能问题 | 前端卡顿 | 低 | 虚拟列表 + 懒加载头像 |
| DeepSeek API 额度/费用超预期 | 成本失控 | 中 | 阶段三加入调用频率限制；缓存相似回复 |

## 技术决策记录

| 决策 | 选项 A | 选项 B | 最终选择 | 理由 |
|------|--------|--------|----------|------|
| 前端框架 | React | Vue 3 | **待定** | 两者生态均满足需求，由前端开发者偏好决定 |
| 状态管理 | Redux | Pinia / Zustand | **待定** | 取决于前端框架选择 |
| 消息推送方式 | WebSocket | SSE | **WebSocket** | 双向通信，支持心跳和状态推送 |
